<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI/service bootstrap -> adapter/runtime/store | The binary must compose existing boundaries correctly instead of bypassing them. |
| readiness/smoke contract -> future external-process phases | Later phases will depend on the exact service startup and probe behavior introduced here. |
| docs -> developers/operators | Ambiguous docs can cause users to run the wrong path (`stress-smoke` instead of `serve`) or bypass architecture rules. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-02 | Tampering | service composition | mitigate | Compose the official service through `adapter_http::router(HttpState)` and existing gateways only. |
| T-11-03 | Denial of Service / Repudiation | service readiness and smoke | mitigate | Add a minimal health route and targeted smoke coverage. |
| T-11-04 | Repudiation | developer/operator guidance | mitigate | Update docs so `app serve` is the official runnable service path and `stress-smoke` is described separately. |
| T-11-05 | Reliability | shared codec/aggregate support | mitigate | Add explicit user aggregate serialization support if required by the serve composition path. |
</threat_model>
