<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| adapter caller -> runtime contracts | Commands, metadata, idempotency keys, and reply channels enter runtime-owned types. |
| runtime -> event store | Runtime submits append and rehydration requests to durable storage. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Denial of Service | CommandEnvelope / gateway contracts | mitigate | Define typed overload and invalid-capacity errors now so later bounded ingress never blocks or silently queues unbounded work. |
| T-03-02 | Elevation of Privilege | CommandEnvelope routing fields | mitigate | Store `CommandMetadata.tenant_id` and computed partition key together in each envelope so later routing can include tenant ID. |
| T-03-03 | Tampering | runtime store boundary | mitigate | Keep append and rehydration behind a trait that forwards to PostgreSQL storage without moving transaction logic into runtime. |
| T-03-04 | Repudiation | CommandOutcome | mitigate | Include `CommittedAppend` in every successful outcome so replies carry durable stream/global positions, not ring sequence numbers. |
| T-03-05 | Tampering | StoreError mapping | mitigate | Map `StoreError::StreamConflict` to structured `RuntimeError::Conflict` for later cache-preserving conflict handling. |
</threat_model>
