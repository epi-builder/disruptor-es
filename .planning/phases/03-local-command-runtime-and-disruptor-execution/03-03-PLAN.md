## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gateway -> disruptor path | Accepted commands cross from bounded Tokio ingress into per-shard in-process sequencing. |
| disruptor path -> shard state | Ring events mutate shard-local handoff/cache state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-10 | Denial of Service | DisruptorPath | mitigate | Use `try_publish` and map `RingBufferFull` to `RuntimeError::ShardOverloaded`; forbid blocking `publish`. |
| T-03-11 | Tampering | AggregateCache | mitigate | Stage and commit cache state through explicit methods; do not expose global mutable cache handles. |
| T-03-12 | Denial of Service | disruptor async bridge | mitigate | Keep async storage out of the sync disruptor handler and grep for `block_on`, nested runtime creation, and `Handle::current`. |
| T-03-13 | Repudiation | disruptor sequence diagnostics | mitigate | Treat disruptor sequence only as local diagnostic handoff order; do not expose it as committed global position. |
</threat_model>
