<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shard state -> durable storage | Decided events cross from in-memory runtime into event-store append. |
| durable storage -> command reply | Committed append positions cross back to runtime callers as authoritative success evidence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-14 | Repudiation | command replies | mitigate | Send success replies only with `CommittedAppend` returned by `RuntimeEventStore::append`. |
| T-03-15 | Tampering | shard cache | mitigate | Stage aggregate state and call `commit_state` only after committed or duplicate append outcomes. |
| T-03-16 | Tampering | OCC conflict path | mitigate | Map `StoreError::StreamConflict` to `RuntimeError::Conflict` and leave cached state unchanged. |
| T-03-17 | Denial of Service | runtime flow | mitigate | Preserve gateway `try_send` overload tests and shard full-ring overload tests from prior plans in the phase gate. |
| T-03-18 | Information Disclosure | tenant-scoped commands | mitigate | Build append requests from `CommandEnvelope.metadata.tenant_id`; do not introduce tenant-free append inputs. |
</threat_model>
