<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Prometheus metrics -> stress report JSON | Observability data crosses from scrape text into archived evidence and must not be mislabeled. |
| CLI/report JSON -> operators and archive summaries | Downstream evidence readers treat these fields as source-of-truth for throughput diagnosis. |
| workload shape config -> request generator | Shape labels must match what the workload really proves. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13.1-04 | T | `crates/app/src/http_stress.rs` report assembly | mitigate | Keep observed scrape fields separate from estimated fallbacks so no synthetic queue-depth value is misreported as observed evidence. |
| T-13.1-05 | I | `crates/app/src/main.rs` JSON output | mitigate | Serialize optional observed fields and explicitly named fallback fields so missing scrapes do not look like zero latency or zero queueing. |
| T-13.1-06 | R | repeated-stream hot-key lanes | mitigate | Add a machine-readable workload-purpose marker so later evidence and docs cannot cite repeated-stream failures as success-throughput baselines. |
</threat_model>
