### Gaps Summary

Phase 13.1 did land the core runtime optimization: the serial dispatcher bottleneck is gone, the `NoStream` fast path exists, and the live HTTP harness now runs real external-process stress with explicit workload shapes and scrape accounting. Those parts are backed by code and passing tests.

The phase still misses full goal achievement because the archived diagnosis evidence is not strong enough to prove the remaining ceiling by layer. The storage-only lane never produced a measurement, live baseline runs had zero successful metric scrapes, `ingress_depth_max` is synthesized when those scrapes fail, and the new runtime runner introduces a shutdown path that can abandon already-accepted commands. The phase therefore improved throughput materially, but it did not close the bottleneck investigation with fully trustworthy and correctness-safe sign-off evidence.
