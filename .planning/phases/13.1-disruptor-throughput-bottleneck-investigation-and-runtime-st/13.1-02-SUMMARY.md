## Decisions Made

- Kept the workload-shape model on `HttpStressConfig` so CLI, benches, and the live runner all use one validated surface.
- Exposed scrape failure counters even when the measured fields remain zero, because the zeroes are only meaningful when at least one scrape succeeds.
- Preserved hot-key/hot-set smoke execution by classifying repeated-stream domain `400 Bad Request` responses as failed outcomes instead of aborting the whole run.

- **Fix:** Added `HttpStressConfig::validate()` checks for `hot_set_size`, including shape compatibility and a bounded `1..=4096` range.

**2. [Rule 1 - Bug] Prevented repeated hot-key domain errors from aborting the harness**
- **Found during:** Task 2
- **Issue:** `SingleHotKey` and `HotSet` runs can legitimately revisit the same order stream; repeated `PlaceOrder` requests returned domain `400` responses and terminated the run before JSON was emitted.
- **Fix:** Counted `400 Bad Request` responses as failed stress outcomes so the harness completes and still reports queueing and scrape diagnostics.
- **Files modified:** `crates/app/src/http_stress.rs`
- **Verification:** `cargo run -q -p app -- http-stress --profile smoke --workload-shape hot-set --hot-set-size 4 --warmup-seconds 1 --measure-seconds 2 --concurrency 2 --command-count 8 --shard-count 2 --ingress-capacity 8 --ring-size 16`
- **Committed in:** `f04f800`
