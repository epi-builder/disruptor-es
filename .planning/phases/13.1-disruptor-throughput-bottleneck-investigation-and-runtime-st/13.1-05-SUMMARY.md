**Observed-only HTTP stress metrics with explicit estimated-ingress fallback and repeat-stream diagnostic labeling**

## Accomplishments
- Added the requested regressions for observed ingress depth, JSON fallback separation, and repeated-stream workload labeling.

- Added `workload_purpose` to both Rust and JSON output so `HotSet` and `SingleHotKey` lanes are explicitly marked `repeat-stream-diagnostic`.

## Files Created/Modified

## Decisions Made

- `ingress_depth_max` is now reserved for observed scrape data only; synthetic fallback values moved to `ingress_depth_estimated_max`.
- Missing scrape-derived queue depth, latency, and lag values remain absent in both Rust and JSON instead of being serialized as fake measurements.
- Repeated-stream order lanes remain executable but are explicitly classified as diagnostic-only evidence.
