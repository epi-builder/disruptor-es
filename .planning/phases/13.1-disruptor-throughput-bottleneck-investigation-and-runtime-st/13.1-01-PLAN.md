<objective>
Refactor the runtime so routed commands fan out to one worker per shard and remove redundant cold-cache reads for `ExpectedRevision::NoStream` commands without violating commit-gated replies.

Purpose: Phase 13.1 research showed the current topology serializes all accepted work through one receiver and one `process_one()` loop, which destroys the throughput benefit expected from per-shard ordered execution.
Output: refactored runtime engine topology, safe no-stream fast path, and focused runtime tests proving shard-parallel execution plus durable reply gating.
</objective>
