<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| adapter ingress -> runtime dispatcher | Accepted user commands cross into bounded internal queues and must receive a determinate outcome during shutdown. |
| runtime dispatcher -> shard workers | Work ownership changes here; shutdown must not silently drop accepted envelopes. |
| shard worker -> durable event store | In-flight commands must stay commit-gated even while the service is stopping. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13.1-01 | D | `crates/es-runtime/src/engine.rs` | mitigate | Keep ingress bounded, stop dispatch on shutdown, and explicitly fail only undispatched accepted commands so queues do not hang indefinitely. |
| T-13.1-02 | T | `crates/es-runtime/src/engine.rs` + shard workers | mitigate | Join shard workers after sender shutdown so already-dispatched commands keep the durable append gate and are not truncated mid-flight. |
| T-13.1-03 | R | `crates/app/src/serve.rs` | mitigate | Cover the official `app serve` shutdown path with runtime-flow regressions tied to the engine runner API used by serve. |
</threat_model>
