<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| committed events table -> projector | Stored event rows and JSON payloads are untrusted until decoded and schema-checked. |
| query caller -> read-model repository | Caller supplies tenant ID, row IDs, optional minimum global position, and wait policy. |
| projection repository -> PostgreSQL | SQL statements write read models and offsets inside durable transactions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-10 | Information Disclosure | read-model queries and offsets | mitigate | Every table primary key and query predicate includes `tenant_id`; integration test `projections_are_scoped_by_tenant` must pass. |
| T-05-11 | Tampering | projection SQL | mitigate | Use static SQL strings with SQLx `.bind(...)` for tenant IDs, projector names, row IDs, positions, statuses, and quantities; no `format!` SQL. |
| T-05-12 | Repudiation | checkpoint correctness | mitigate | Persist `projector_offsets.last_global_position` in the same transaction as read-model upserts; tests verify offset equals the last applied committed event. |
| T-05-13 | Denial of Service | malformed payload catch-up | mitigate | Convert handled-event decode failures to `ProjectionError::PayloadDecode` and do not advance offset, preventing silent skip of poisoned data. |
| T-05-14 | Denial of Service | read-your-own-write query wait | mitigate | Delegate to Plan 01 `wait_for_minimum_position`; lagging queries return `ProjectionError::ProjectionLag` after `WaitPolicy.timeout`. |
| T-05-15 | Tampering | invalid cursor/limit inputs | mitigate | Accept only `MinimumGlobalPosition` and `ProjectionBatchLimit`; negative positions and invalid limits are rejected before SQL execution. |
</threat_model>
