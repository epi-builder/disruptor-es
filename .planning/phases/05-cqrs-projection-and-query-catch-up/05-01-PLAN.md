<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| query caller -> projection API | Caller supplies projector names, minimum global positions, batch limits, and wait-policy values. |
| projection API -> storage implementation | Validated projection contracts are passed to PostgreSQL implementation in later plans. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | `ProjectorOffset` / `MinimumGlobalPosition` contracts | mitigate | Require `ProjectorOffset` to carry `TenantId`; downstream storage must key offsets by `(tenant_id, projector_name)` and never use global-only checkpoints. |
| T-05-02 | Denial of Service | `wait_for_minimum_position` | mitigate | Enforce nonzero polling interval, finite timeout, and return `ProjectionError::ProjectionLag` instead of unbounded loops. |
| T-05-03 | Tampering | cursor and limit constructors | mitigate | Reject negative global positions and invalid batch limits before any SQL call. |
| T-05-04 | Denial of Service | malformed event payload path | mitigate | Define `ProjectionError::PayloadDecode { event_type, schema_version }`; downstream projectors must return this error and avoid advancing offsets for failed events. |
| T-05-05 | Tampering | future SQL use of projection values | mitigate | Plan 03 must use SQLx `.bind(...)` for projector names, tenant IDs, positions, and read-model IDs; this plan supplies typed values only. |
</threat_model>
