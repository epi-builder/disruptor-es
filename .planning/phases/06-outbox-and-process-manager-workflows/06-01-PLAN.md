<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| append request -> outbox contracts | Untrusted integration message fields enter typed construction before storage. |
| dispatcher -> publisher | Durable rows become external effects through a publisher trait. |
| process manager -> command gateway | Replayable events will later create follow-up command envelopes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | `OutboxMessage.tenant_id` | mitigate | Require `tenant_id: TenantId` on `OutboxMessage`; no constructor accepts an empty tenant because `TenantId::new` rejects empty values. |
| T-06-02 | Tampering | `OutboxMessage::idempotency_key` | mitigate | Build idempotency key exactly from `{tenant_id}:{topic}:{source_event_id}` and test repeated publishes through `InMemoryPublisher`. |
| T-06-03 | Denial of Service | `RetryPolicy` and future dispatcher | mitigate | Validate `RetryPolicy::new(max_attempts)` with `max_attempts >= 1`; downstream dispatcher must mark rows failed after this bound. |
| T-06-04 | Tampering | SQL construction | accept | This plan adds no SQL; later PostgreSQL plans must use SQLx bind parameters only. |
| T-06-05 | Tampering | Process-manager replay keys | mitigate | Define `ProcessManagerName` and keep deterministic key components available for Plan 05; do not add random or time-based workflow idempotency keys. |
</threat_model>
