<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Runtime shard cache -> command caller | Shard-local replay records are used to answer duplicate caller requests. |
| PostgreSQL command_dedup -> runtime shard | Durable replay records cross from JSONB storage into typed runtime replies. |
| Domain aggregate -> runtime idempotency layer | Idempotency decisions must intercept commands before domain state is read or mutated. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | `ShardState::process_next_handoff` | mitigate | Local and durable dedupe branches execute before `rehydrate_state` and `A::decide`; runtime tests assert no append occurs on cache/store hits. |
| T-08-02 | Tampering | `DedupeKey` and runtime lookup | mitigate | `DedupeKey` keeps `tenant_id` plus `idempotency_key`; durable lookup receives both fields from `CommandMetadata` and envelope idempotency key. |
| T-08-03 | Tampering / Denial of Service | `RuntimeEventCodec::decode_reply` | mitigate | Runtime decodes stored reply payloads through codec-owned typed validation and returns `RuntimeError::Codec` for mismatched replay payloads. |
| T-08-04 | Tampering | duplicate append race branch | mitigate | `AppendOutcome::Duplicate` branch performs a durable replay lookup and never sends `decision.reply`, preventing duplicate follow-up side effects from fresh decisions. |
</threat_model>
