<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| runtime cache -> command reply | Shard-local dedupe cache payload is decoded into a typed reply. |
| runtime -> PostgreSQL | Shard runtime performs durable duplicate lookup before aggregate state load. |
| PostgreSQL -> runtime codec | Stored JSON reply payload crosses into typed aggregate reply reconstruction. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing/Tampering | `DedupeKey` and store lookup | mitigate | Construct `DedupeKey` only from `envelope.metadata.tenant_id` and `envelope.idempotency_key`; store lookup uses the same pair. |
| T-08-02 | Tampering | `RuntimeEventCodec::decode_reply` | mitigate | Duplicate replay decodes persisted original `reply_payload`; no duplicate branch calls `A::decide` to recompute reply. |
| T-08-03 | Elevation of privilege | adapter-local retry state | transfer | No adapter changes in this plan; runtime becomes the shared replay enforcement point used by all gateways. |
| T-08-04 | Repudiation/DoS | process-manager repeated follow-ups | mitigate | Runtime replay handles deterministic process-manager idempotency keys the same as HTTP retries. |
</threat_model>
