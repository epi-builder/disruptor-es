<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client -> adapter | Untrusted JSON request supplies tenant, idempotency key, command IDs, and correlation ID. |
| adapter -> runtime | Adapter submits commands through `CommandGateway` and awaits runtime replies. |
| process manager -> runtime | Workflow replay resubmits deterministic follow-up commands after crash/retry. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing/Tampering | HTTP and process-manager idempotency keys | mitigate | Tests require same tenant/idempotency key to replay and different tenant behavior remains store/runtime scoped. |
| T-08-02 | Tampering | HTTP duplicate response body | mitigate | Adapter test verifies duplicate response uses original committed append/reply fields, not mutated-state recomputation. |
| T-08-03 | Elevation of privilege | HTTP adapter | mitigate | Tests must not add adapter-local maps or direct `PostgresEventStore`; duplicates flow through `CommandGateway`. |
| T-08-04 | Repudiation/DoS | process-manager crash/retry | mitigate | App test replays the same `ProcessEvent` and validates deterministic follow-up keys plus successful duplicate replies. |
</threat_model>
