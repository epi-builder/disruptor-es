## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| runtime -> store | Runtime supplies tenant/idempotency lookup values and response payload JSON to PostgreSQL. |
| database -> runtime | Runtime decodes persisted duplicate response payloads from `command_dedup`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing/Tampering | `command_dedup` lookup | mitigate | All lookup and cache inputs use `(tenant_id, idempotency_key)`; SQL predicates bind both values. |
| T-08-02 | Tampering | `response_payload` decode | mitigate | Persist `StoredCommandOutcome` and decode that exact JSON instead of recomputing from aggregate state. |
| T-08-03 | Elevation of privilege | HTTP adapter retry state | transfer | Plan 03 verifies adapters remain thin; this plan exposes only store/runtime APIs. |
| T-08-04 | Tampering/Repudiation | process-manager retry | transfer | Plan 03 verifies deterministic process-manager keys replay through runtime/store duplicate lookup. |
</threat_model>
