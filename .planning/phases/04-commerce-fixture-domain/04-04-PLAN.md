<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| command caller -> `Order::decide` | Untrusted order lines and relationship assumptions enter domain validation. |
| order aggregate -> product/user aggregates | Order stores IDs and assumption snapshots only; it must not mutate or embed other aggregate state. |
| generated tests -> aggregate invariants | Randomized command sequences attempt invalid states and must be rejected by typed errors. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-14 | Tampering | `PlaceOrder` | mitigate | Reject empty orders, inactive users, and unavailable products before emitting `OrderPlaced`. |
| T-04-15 | Tampering | duplicate placement | mitigate | Return `OrderError::AlreadyPlaced` when state is not `OrderStatus::Draft`. |
| T-04-16 | Tampering | terminal transitions | mitigate | Return `OrderError::AlreadyTerminal` for confirm/reject/cancel after confirmed, rejected, or cancelled state. |
| T-04-17 | Information Disclosure | cross-tenant leakage through shared state | mitigate | Keep order state per aggregate instance and assert no `UserState` or `ProductState` object references are stored in `order.rs`. |
| T-04-18 | Elevation of Privilege | dependency-boundary regression | mitigate | Full workspace tests include existing dependency boundary checks; this plan adds no runtime/storage/adapter dependency to `example-commerce`. |
</threat_model>
