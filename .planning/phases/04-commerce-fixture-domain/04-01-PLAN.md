<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| command caller -> commerce constructors | Untrusted caller strings and numeric quantities become typed domain IDs and quantities. |
| commerce facade -> later aggregate modules | Public exports define which domain types runtime, projection, and process-manager phases may depend on. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | `ids.rs` constructors | mitigate | Reject empty `UserId`, `ProductId`, `OrderId`, and `Sku` values with `CommerceIdError::EmptyValue`; reject zero `Quantity` with `CommerceIdError::InvalidQuantity`. |
| T-04-02 | Tampering | `Quantity` public API | mitigate | Expose only `Quantity::new(u32)` and `Quantity::value()`, avoiding signed public quantities that could bypass inventory checks later. |
| T-04-03 | Information Disclosure | module facade | accept | Phase 04 contains no tenant data store and no query/API surface; cross-tenant leakage is handled later by runtime/store metadata, but this plan avoids global mutable business state. |
| T-04-04 | Elevation of Privilege | dependency boundary | mitigate | Keep `example-commerce` dependencies limited to `es-core`, `es-kernel`, and `thiserror`; no `tokio`, `sqlx`, adapters, brokers, or `disruptor` dependency is added. |
</threat_model>
