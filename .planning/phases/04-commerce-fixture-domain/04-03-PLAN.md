<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| command caller -> `Product::decide` | Untrusted inventory deltas and quantities enter aggregate-local validation. |
| product aggregate -> future order/process-manager logic | Product availability becomes the source for later order relationship assumptions and inventory workflow commands. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-09 | Tampering | `AdjustInventory` | mitigate | Compute `state.available_quantity + delta` with explicit validation and return `InventoryWouldBeNegative` before emitting an event. |
| T-04-10 | Tampering | `ReserveInventory` | mitigate | Reject reservations where `quantity.value() as i32 > available_quantity` with `InsufficientInventory`. |
| T-04-11 | Tampering | `ReleaseInventory` | mitigate | Reject releases where `quantity.value() as i32 > reserved_quantity` with `InsufficientReservedInventory`. |
| T-04-12 | Denial of Service | inventory arithmetic | mitigate | Avoid saturating/clamping arithmetic that hides invalid commands; property tests assert nonnegative quantities after generated command sequences. |
| T-04-13 | Information Disclosure | cross-tenant leakage through shared state | accept | Product state remains per aggregate instance and has no static/global inventory maps. |
</threat_model>
