<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user CLI flags -> live stress runner | Invalid or misleading CLI inputs can create unsafe runs or confusing output. |
| report JSON/docs -> archive-facing evidence | The output must distinguish steady-state evidence from Phase 12 smoke and Criterion behavior. |
| benchmark wrapper -> steady-state runner | Divergent benchmark logic would create multiple incompatible external HTTP measurement paths. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-05 | Tampering | `app http-stress` CLI surface | mitigate | Support only explicit local sizing/profile flags, reject unknown values, and omit arbitrary target URL support. |
| T-13-06 | Information Disclosure | JSON report and docs | mitigate | Print only bounded run metadata and required report keys; do not echo `DATABASE_URL`, environment dumps, or secret values in JSON or docs. |
| T-13-07 | Repudiation | docs and benchmark labels | mitigate | Mark Criterion as secondary smoke-only evidence and document Phase 13 JSON output as the steady-state source of truth. |
| T-13-08 | Reliability | benchmark wrapper | mitigate | Keep `benches/external_process_http.rs` delegating to the shared validated config/runner path instead of forking new process orchestration code. |
</threat_model>
