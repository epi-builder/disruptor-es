## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer machine -> Cargo registry | Workspace resolves third-party crates from registries and package sources. |
| workspace policy -> member crates | Root dependency and lint rules must constrain lower-level crates without silently adding runtime dependencies. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `Cargo.toml`, `deny.toml` | mitigate | Pin explicit workspace dependency versions, set `resolver = "3"`, and add `deny.toml` with unknown registry/git denied. |
| T-01-02 | Elevation of Privilege | Workspace Rust lints | mitigate | Set `[workspace.lints.rust] unsafe_code = "forbid"` and require member crates to inherit workspace lints in later plans. |
| T-01-03 | Information Disclosure | Metadata dependency catalog | accept | Phase 1 has no runtime secrets or user data; metadata structs are typed in later plans and do not include arbitrary request payloads. |
</threat_model>
